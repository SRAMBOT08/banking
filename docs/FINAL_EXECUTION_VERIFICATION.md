## 3. Payload Flow

The payload generated by `PayloadBuilder` is compatible with the ServiceNow
Table API create endpoint.

### Payload Fields

- `table`: internal routing field used by the executor
- `short_description`: required ServiceNow incident field
- `description`: required ServiceNow incident field
- `priority`: optional, normalized string
- `urgency`: optional, normalized string
- `impact`: optional, normalized string
- `category`: optional, normalized string
- `assignment_group`: optional human-readable reference value
- `caller`: optional human-readable reference value

### Compatibility Review

- The executor sends `IncidentPayload` as JSON to `POST /api/now/table/incident`.
- Required incident content is present:
  - `short_description`
  - `description`
- Optional fields are serialized safely as strings or omitted as `None` values in the dataclass-derived payload.
- Reference-like fields remain human-readable by design and are not converted to `sys_id`.
- The payload shape is compatible with the current MVP architecture and the ServiceNow Table API create flow.

## 5. Error Flow

```text
Failure at any stage
    |
    v

    |
    v

  - stage
  - correlation_id
  - exception type
  - exception message
  - stack trace
    |
    v
Raised exception or VerificationResult(verified=False)
```

### Review

- Runtime initialization logs failures and re-raises.
- Pipeline failures are converted to `VerificationResult(verified=False, ...)`.
- Incident service logs and re-raises orchestration failures.
- Payload builder, table executor, client, and verifier all emit stage-level logs on failure paths.
- No error path was found that silently skips logging in the Create Incident flow.

## 6. Correlation ID Flow

The same correlation ID is threaded through the complete execution path:

1. Created in the smoke script or pipeline entrypoint
2. Passed into `ServiceNowPipeline.execute_create_incident(...)`
3. Passed into `IncidentService.create_incident(...)`
4. Passed into `PayloadBuilder.build_incident_payload(...)`
5. Passed into `TableExecutor.create_record(...)`
6. Passed into `ServiceNowClient.request(...)`
7. Reused in the HTTP response log
8. Passed into `ExecutionVerifier.verify_creation(...)`
9. Reused in final completion logs

This is sufficient to trace a single execution across all layers.
