# Folder Structure

- `app/api`: REST routes and request schemas
- `app/config`: settings
- `app/core`: logging
- `app/models`: domain models
- `app/policies`: pluggable deterministic policy framework
- `app/queue`: deterministic queue engine
- `app/services`: planner, orchestrator, verification, audit
- `app/events`: Kafka consumer/publisher/pipeline
- `app/repositories`: in-memory state and audit storage
- `tests`: unit/integration coverage
- `docs`: architecture and operations guides
